  • Zufällige Farbe generieren

  • Abgleichen ob diese Farbe schon zweimal abgespeichert wurde, falls ja neue Farbe generieren.
    Wenn nicht: abspeichern und bei samecolor plus eins rechenen.

  • Gibt generierte Farben zurück in den Code um dann den User input zu korriegieren.

  • Methode generiert zufällige Zahl und wird in der Methode oben genutzt um zufällige Farben zu generieren

  • Methode startagain fragt den Spieler ob er nochmals spielen will.
    Der Scanner erkennt die Antwort. Ist die Antwort falsch beendet er den Code, bei ja startet der Code von vorne.

  • Die Anzahl Farben an richtiger Stelle wird überprüfet. Ist die Zahl gleich 4 gibt es aus, dass der Spieler gewonnen hat.
    Falls nicht wird die Anzahl Runden überprüft, und bei 12 Runden gibt die Methode aus das der Spieler verloren hat.

  • Die Methode wrongPlace kontrolliert ob die im User input gespeicherten Farben richtige Farben sind,
    Bei ja wird die Farbe in den kopierten generierten Farben überschrieben. Das ist, damit keine Zahl doppelt gezählt wird.

  • Die Methode correctPlace kontrolliert ob die im User input eingegebenen Farben an der richtigen Stellen stehen.
    Falls ja wird die kopierte Farbe überschrieben, damit keine Zahl doppelt gezählt wird oder beim abgleichen,
    mit der Methode wrongPlace nicht nochmals gezählt wird.

  • Methode colorReader bittet den Spieler zuerst eine Farbe einzugeben.
    Anschliessend wird die eingegebene Farbe im User input mit jeder Stelle (s) des Arrays color abgeglichen ob die Farbe überhaupt existiert.
    Dabei wird nicht auf die Gross- und Kleinschreibung geachtet und jeder Abstand wird ignoriert. Somit werden nur richtige Farben angenomen
    Der Spieler kann nicht eine ungültige Farbe einloggen. Bei falschen Farben wird der Spieler per Fehlermeldung informiert, dass die eingegebene Farbe nicht existiert.
