  • gibt generierte Farben zurück nach oben in den Code um nacher den User input zu korriegieren.

  • Methode generiert Zufällige Zahl und wird in der Methode oben an genutzt um zufällige Farbe zu generieren

  • Methode startagain fragt den Spieler ob er nochmals spielen will.
    Der Scanner erkennt die Antwort bei nein beendet er den Code bei ja startet der code von vorne.

  • Die anzahl farben an richtiger stelle wird überprüfed, ist die zahl gleich 4 gibt es aus das der Spieler gewonnen hat.
    falls nicht wird die anzahl runden überprüft bei 12 runden gibt die Methode aus das der Spieler verloren hat.

  • Die Methode wrongPlace kontrolliert ob die im User input gespeicherten Farben richtige Farben sind,
    falls ja dann überschreibt es die Farbe in den kopierten generierten Farben, damit keine Zahl doppelt gezählt wird.

  • Die Methode correctPlace kontroliert ob die im User input eingegebenen Farben an der richtigen stellen stehen.
    Falls ja wird die kopierte Farbe überschrieben, damit keine Zahl doppelt gezählt wird oder beim abgleichen,
    mit der Methode wrongPlace nicht nochmals gezählt wird.

  • Methode colorReader bittet den Spieler zuerst eine Farbe einzugeben.
    Anschliessend wird die eingegebene Farbe im User input mit jeder Stelle (s) des Arrays color abgeglichen ob die Farbe überhaupt existiert.
    dabei wird nicht auf die Gross- und Kleinschreibung geachtet. Und jeder Abstand wird ignoriert, somit werden nur richtige Farben angenomen
    und der Spieler kann nicht eine ungültige Farbe einloggen. Bei falschen Farben wird der Spieler per Fehlermeldung informiert, dass die eingegebene Farbe nicht existiert.
